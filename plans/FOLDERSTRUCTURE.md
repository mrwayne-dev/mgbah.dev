### `.htaccess`
Apache config. Two jobs: (1) route all non-file requests to `index.php` for SPA routing, (2) security hardening.
```apache
RewriteEngine On
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^ index.php [QSA,L]

# Block direct PHP file access outside api/
<FilesMatch "\.php$">
  Order Deny,Allow
  Deny from all
</FilesMatch>
<FilesMatch "^index\.php$">
  Allow from all
</FilesMatch>
<Files "api/contact.php">
  Allow from all
</Files>

# Compression
<IfModule mod_deflate.c>
  AddOutputFilterByType DEFLATE text/html text/css application/javascript
</IfModule>

# Cache static assets
<IfModule mod_expires.c>
  ExpiresActive On
  ExpiresByType image/webp "access plus 1 year"
  ExpiresByType font/woff2 "access plus 1 year"
  ExpiresByType text/css "access plus 1 month"
  ExpiresByType application/javascript "access plus 1 month"
</IfModule>
```

### `index.php`
**The single entry point for the entire site.** Serves the HTML shell that the SPA runs inside.

Responsibilities:
- Load `config/env.php` to make environment variables available
- Output the full HTML document with `<head>` (meta, fonts, CSS, CDN scripts) and `<body>` (nav placeholder, `<main id="app">`, footer placeholder)
- Include the Phosphor Icons CDN script
- Include GSAP CDN script
- Include tsParticles CDN script
- Include Cal.com embed script
- Load `assets/js/app.js` as a module at bottom of body

## `/api`

### `api/contact.php`
The only publicly accessible API endpoint. Handles the contact form submission.

Responsibilities:
1. Require `includes/headers.php` (sets CORS + Content-Type headers)
2. Reject non-POST requests
3. Require `includes/rate_limit.php` — abort if rate limited
4. Parse and sanitise input via `includes/helpers.php`
5. Validate: name required, email valid format, message not empty
6. Require `includes/mailer.php` — call `sendMail()` to notify Michael
7. Call `sendAutoReply()` to email the sender using `mail_templates/contact_autoreply.html`
8. Return JSON via `includes/responses.php`

### `api/email_templates.php`
PHP functions that build the HTML email body sent to Michael when someone contacts him. Not the auto-reply — that's the HTML template. This generates the notification email.

Example:
```php
function buildContactNotification($name, $email, $message): string {
    return "
      <h2>New message from {$name}</h2>
      <p><strong>Email:</strong> {$email}</p>
      <p><strong>Message:</strong><br>{$message}</p>
    ";
}
```

### `api/mail_templates/contact_autoreply.html`
Standalone HTML email template sent to the person who contacted Michael. Confirms receipt, sets expectations. Styled dark, minimal. References Michael's name and possibly Lymora. Should feel personal, not automated.

## `/assets/js`

### `app.js`
The entry point. Imported as a module in `index.php`. Runs once on load.

Responsibilities:
- Import and initialise the router (`router.js`)
- Import and mount the nav component (`components/nav.js`)
- Import and mount the chat widget (`components/chat.js`)
- Import and initialise particles on the home page (`components/particles.js`)
- Set up global Intersection Observer for scroll animations
- Start the initial route render

```js
import { Router } from './router.js';
import { initNav } from './components/nav.js';
import { initChat } from './components/chat.js';
import { initScrollAnimations } from './utils/transitions.js';

initNav();
initChat();
initScrollAnimations();

const router = new Router();
router.init();
```

### `router.js`
The SPA router. Intercepts all internal link clicks, uses `history.pushState()` to update the URL without reload, and renders the correct page module into `<main id="app">`.

Responsibilities:
- Define route map: `{ '/': homePage, '/about': aboutPage, '/projects': projectsPage, '/lymora': lymoraPage, '/contact': contactPage }`
- Intercept `<a>` clicks that are internal links
- Call `history.pushState()` with the new path
- Fetch the page module and call its render function
- Run page exit transition before swap, page enter transition after
- Update `<title>` and meta description on each navigation
- Handle `popstate` event for browser back/forward buttons
- Handle 404 — render a minimal not-found state

```js
export class Router {
  constructor() {
    this.routes = {
      '/':         () => import('./pages/home.js'),
      '/about':    () => import('./pages/about.js'),
      '/projects': () => import('./pages/projects.js'),
      '/lymora':   () => import('./pages/lymora.js'),
      '/contact':  () => import('./pages/contact.js'),
    };
  }

  async navigate(path) {
    await pageExit(); // transition out
    const mod = await this.routes[path]?.();
    document.getElementById('app').innerHTML = mod.render();
    await pageEnter(); // transition in
    mod.init?.(); // run page-specific JS after render
  }
}
```

## `/assets/js/components`

### `nav.js`
Renders the navigation HTML into `<nav id="nav">` and attaches behaviour.

- Marks active link based on current `window.location.pathname`
- Listens for router navigation events to update active state
- Scroll listener: adds `.scrolled` class to nav when page scrolls past 50px (triggers blur + border)
- Mobile: toggles `.nav-open` class on `body` for overlay menu
- Manages focus trap inside mobile menu for accessibility

### `particles.js`
Initialises tsParticles on the home page hero canvas.

- Only runs when the home page is rendered (checked via DOM presence of `#hero-canvas`)
- Exports `initParticles()` called by `home.js` after render
- Config: slow-moving, very low opacity white particles. Smoke, not sparkles.
- Destroys the instance on page navigation to prevent memory leaks

### `chat.js`
The WhatsApp live chat widget. Mounted once in `app.js`, persists across all page navigations.

- Renders the bubble and panel HTML into `<div id="chat-widget">`
- Bubble: floating button, bottom-right, always visible
- Panel: chat window with greeting + input + send button
- On send: constructs WhatsApp deep link and opens in new tab
- Auto-open: after 45s if not dismissed in this session
- Session state: saves `chat_dismissed` to `sessionStorage`
- WhatsApp number read from a constant (set in `config/constants.php` and passed via a `<meta>` tag or inline JS variable in `index.php`)

## `/assets/js/pages`

Each page module exports two things:
- `render()` → returns an HTML string to inject into `<main id="app">`
- `init()` → runs after the HTML is in the DOM (attaches event listeners, triggers animations, initialises page-specific features)

### `home.js`
Renders the home page. Calls `initParticles()` from `particles.js` in its `init()`. Sets up Intersection Observer for the selected work cards and about strip. Handles Cal.com button init.

### `about.js`
Renders the About page. Two-column layout. `init()` triggers scroll entrance animations on skills pills and experience rows.

### `projects.js`
Renders the Projects page. `init()` sets up filter tab click handlers to show/hide project cards by category. May also handle project modal/expand logic.

### `lymora.js`
Renders the Lymora dedicated page. `init()` triggers counter animations on the traction numbers (e.g. counting up to 100+, 500k+).

### `contact.js`
Renders the Contact page. `init()` responsibilities:
- Sets up contact form submit handler (async fetch to `api/contact.php`)
- Handles success/error state display inline
- Initialises Cal.com embed button
- Re-initialises chat widget prompt if needed

## `/assets/js/utils`

### `transitions.js`
Page transition and scroll animation utilities.

Exports:
- `pageExit()` → animates current page content out (opacity 0, slight translate down). Returns a Promise that resolves after animation.
- `pageEnter()` → animates new page content in (opacity 1, translate to 0). Returns a Promise.
- `initScrollAnimations()` → sets up a global Intersection Observer that watches for `.fade-up` and `.fade-in` elements and adds `.is-visible` class when they enter the viewport
- Uses GSAP for smooth, controllable transitions

### `env.php`
Loads `.env` file variables into PHP environment. Called first in `index.php` and `api/contact.php`.
```php
<?php
$envFile = __DIR__ . '/../.env';
if (file_exists($envFile)) {
    $lines = file($envFile, FILE_IGNORE_NEW_LINES | FILE_SKIP_EMPTY_LINES);
    foreach ($lines as $line) {
        if (strpos($line, '#') === 0) continue;
        [$key, $value] = explode('=', $line, 2);
        putenv(trim($key) . '=' . trim($value));
    }
}
```

### `helpers.php`
Reusable utility functions. Sanitisation, validation, formatting.
```php
<?php
function sanitize(string $input): string {
    return htmlspecialchars(strip_tags(trim($input)), ENT_QUOTES, 'UTF-8');
}

function isValidEmail(string $email): bool {
    return filter_var($email, FILTER_VALIDATE_EMAIL) !== false;
}

function isEmpty(string $value): bool {
    return trim($value) === '';
}
```

### `mailer.php`
PHPMailer initialisation wrapper. Sets up the SMTP connection once and exposes clean `sendMail()` and `sendAutoReply()` functions. `contact.php` calls these — never touches PHPMailer directly.

```php
<?php
use PHPMailer\PHPMailer\PHPMailer;
use PHPMailer\PHPMailer\Exception;

require_once __DIR__ . '/../vendor/autoload.php';
require_once __DIR__ . '/../config/env.php';

function createMailer(): PHPMailer {
    $mail = new PHPMailer(true);
    $mail->isSMTP();
    $mail->Host       = getenv('SMTP_HOST');
    $mail->SMTPAuth   = true;
    $mail->Username   = getenv('SMTP_USER');
    $mail->Password   = getenv('SMTP_PASS');
    $mail->SMTPSecure = PHPMailer::ENCRYPTION_STARTTLS;
    $mail->Port       = getenv('SMTP_PORT');
    $mail->setFrom(getenv('SMTP_USER'), getenv('SMTP_FROM_NAME'));
    return $mail;
}

function sendMail(string $subject, string $htmlBody): bool { ... }
function sendAutoReply(string $toEmail, string $toName): bool { ... }
```

### `rate_limit.php`
Simple file-based or session-based rate limiter. Prevents contact form spam.

Logic:
- Key: hash of sender IP address
- On each request: check how many submissions from this IP in the last `RATE_LIMIT_WINDOW` seconds
- If over `RATE_LIMIT_MAX`: call `responses.php` error and `exit`
- If under: log the request and proceed

### `responses.php`
Standardised JSON response helper. Ensures every API response has the same shape.
```php
<?php
function success(string $message, array $data = []): void {
    echo json_encode(['status' => 'success', 'message' => $message, 'data' => $data]);
    exit;
}

function error(string $message, int $code = 400): void {
    http_response_code($code);
    echo json_encode(['status' => 'error', 'message' => $message]);
    exit;
}
```
